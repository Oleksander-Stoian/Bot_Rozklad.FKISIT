# Telegram Bot: Розклад Коледжу

Цей телеграм-бот призначений для зручного перегляду розкладу занять студентами та викладачами коледжу. Бот підтримує сповіщення про пари, автоматичне визначення чисельника/знаменника тижня та роботу з групами.

## Основні можливості

- **Перегляд розкладу:**
  - "Яка зараз пара?" (з відображенням часу до кінця/початку).
  - Розклад на сьогодні.
  - Розклад на весь тиждень.
  - Розклад дзвінків.
- **Ролі:** Підтримка режиму студента (вибір групи) та викладача (пошук за прізвищем).
- **Сповіщення:**
  - Автоматичні нагадування перед початком пар.
  - Можливість налаштування підписки на сповіщення через меню.
- **Адміністрування:**
  - Команда `/reload_schedule` для оновлення кешу розкладу без перезапуску бота (доступно тільки адміністраторам).
- **Продуктивність:**
  - Кешування даних розкладу (LRU Cache).
  - Використання Redis для зберігання станів користувачів та налаштувань.
  - Асинхронна архітектура (Aiogram 3).

## Технологічний стек

- **Python 3.11**
- **Aiogram 3.x** (Асинхронний фреймворк для Telegram API)
- **Redis** (Зберігання сесій, налаштувань та кешування)
- **Pandas & OpenPyXL** (Обробка Excel-файлів розкладу)
- **Docker & Docker Compose** (Контейнеризація та розгортання)

## Встановлення та запуск

### Варіант 1: Запуск через Docker (Рекомендовано)

Це найпростіший спосіб запустити бота разом з базою даних Redis.

1.  **Клонуйте репозиторій:**
    ```bash
    git clone <repository-url>
    cd <project-folder>
    ```

2.  **Створіть файл `.env`:**
    Створіть файл `.env` у корені проєкту та додайте туди ваші налаштування (див. приклад нижче).

3.  **Запустіть контейнери:**
    ```bash
    docker-compose up -d --build
    ```

### Варіант 2: Локальний запуск (без Docker)

1.  **Встановіть Python 3.11+**
2.  **Встановіть та запустіть Redis** (стандартний порт 6379).
3.  **Встановіть залежності:**
    ```bash
    pip install -r requirements.txt
    ```
4.  **Створіть файл `.env`** (див. нижче).
5.  **Запустіть бота:**
    ```bash
    python main.py
    ```

## Конфігурація (.env)

Створіть файл `.env` у кореневій директорії проєкту та заповніть його:

```env
# Токен вашого бота від @BotFather
BOT_TOKEN=your_telegram_bot_token

# Налаштування Redis (за замовчуванням для Docker)
REDIS_HOST=redis
REDIS_PORT=6379

# ID адміністраторів (через кому, без пробілів)
ADMIN_IDS=123456789,987654321
```

> **Важливо:** Якщо ви запускаєте бота локально (без Docker), змініть `REDIS_HOST=localhost`.


## Оновлення розкладу

Розклад зчитується з файлу `rozklad_pro.xlsx`. Оскільки файл змінюється рідко, він кешується в пам'яті.
Щоб оновити розклад після заміни файлу `.xlsx`:
1.  Замініть файл `rozklad_pro.xlsx` на сервері.
2.  Надішліть боту команду `/reload_schedule` (тільки для адміністраторів).
